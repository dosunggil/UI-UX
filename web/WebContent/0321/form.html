입력양식실습

form 태그 : 사용자의 입력을 받는 화면을 구성할 때 사용하는 태그로,
입력한 데이터를 다른 페이지로 보낼 수 있는 태그입니다.
예) 로그인할 때 아이디와 비밀번호를 입력.
회원가입할 때 가입에 필요한 여러 정보를 입력.

input 태그

text 한 줄 텍스트를 입력하는 태그 (여러줄은 안됩니다.)
password 입력 텍스트가 보이지 않는 비밀번호 태그
hidden 실제 소스에는 있지만 화면에서는 보이지 않는 히든 태그
radio 여러항목 중에 한 개만 선택할 때 사용하는 태그
checkbox 여러항목을 모두 선택할 수 있는 태그
file 첨부할 파일을 선택하는 태그
submit 폼 태그내의 입력데이터를 전송하는 버튼입니다.


**form 태그를 통해 입력 데이터를 전송하려면 반드시 name 속성이 있어야 합니다.

**action 속성값이 없거나 지정하지 않으면 자기 자신 페이지를 연결합니다.

**autocomplete 속성은 자동완성 기능 ( 없으면 기본으로 on )

**전송되는 입력값들을 파라미터라고 합니다.
전송하는 데이터가 여러개인 경우 각 파라미터는 & 로 구분되어 있습니다.
파라미터가 노출이 되는 형태, 노출이 되지 않는 형태가 있는데,
노출되는 형태는 method: get 방식(기본값), 노출되지 않는 형태는 method: post 방식

**placeholder : 텍스트 입력태그에 무엇을 입력하는 태그인지 보여지게 하는 속성

**required : 반드시 입력해야만 하는 필수입력항목을 입력하지 않으면 submit 되지 않도록 처리하려면
프로그래밍적으로 해결할 수도 있지만 태그 자체의 속성을 이용할 수도 있다.

**readonly : 입력태그이지만 변경 불가

**disalbed : 입력태그이지만 변경 불가

*차이점 : readonly 태그는 submit 시에 데이터가 전송됩니다.
disabled 태그는 submit 시에 데이터가 전송되지 않습니다.

**파일을 첨부하여 전송하는 경우 반드시 지정해야 할 속성
1.method 를 post 로 지정해야 합니다.
2.enctype 을 multipart/form-data 를 지정해야만 합니다.
